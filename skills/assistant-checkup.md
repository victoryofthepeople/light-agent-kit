# Assistant Checkup

## Purpose

Review whether the Lumina assistant is set up clearly, safely, and usefully.

## Use When

- The user finishes onboarding.
- The user adds new folders or tools.
- The assistant starts feeling messy.
- The user plans to publish or share anything.

## Read First

- `home.md`
- `POWER-CATALOG.md`
- `POWER-RECIPES.md`
- `profile/me.md`
- `profile/goals.md`
- `profile/preferences.md`
- `profile/taste-and-standards.md`
- `profile/capability-map.md`
- `profile/power-map.md`
- `profile/permission-map.md`
- `power/`
- `tasks/active.md`
- `tasks/lessons.md`
- `PRIVACY-CHECKLIST.md`

## Steps

1. Check whether `home.md` gives a true current picture.
2. Check whether the assistant knows the user's current goals.
3. Check whether communication style and quality standards are clear.
4. Check whether approval gates are clear.
5. Check whether the capability map matches what is actually connected now.
6. Check whether the power map has a clear single Next Activation.
7. Check whether any active power has a matching activation card in `power/`.
8. Check whether active tasks are in one place.
9. Check whether lessons exist and are usable.
10. Check whether private information is staying private.
11. Flag any skill or recent action that wrote outside its `permission-map.md` access level.
12. Suggest the next useful improvement.

## Rules

- Keep the check plain and practical.
- Prefer one useful fix over a long audit.
- Use the privacy checklist before public sharing.
- Re-run this after adding phone access, app access, skills, or a runtime.
