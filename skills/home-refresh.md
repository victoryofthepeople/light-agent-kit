# Home Refresh

## Purpose

Keep `home.md` useful as the first visible page of the Lumina workspace.

It should show what matters now, what the assistant knows, what it can do, what needs approval, and what power may come next.

## Use When

- During onboarding after the first useful result is clear.
- After capability discovery.
- After a project, goal, permission, or power map changes.
- When the user asks "where are we?" or "what can you do for me right now?"

## Read First

- `home.md`
- `profile/me.md`
- `profile/goals.md`
- `profile/preferences.md`
- `profile/taste-and-standards.md`
- `profile/capability-map.md`
- `profile/power-map.md`
- `profile/permission-map.md`
- `power/`
- `tasks/active.md`
- today's `daily/` file, if it exists
- relevant `projects/` files

## Steps

1. Find the current focus from goals, active tasks, today's note, and current project files.
2. Write a short "What The Assistant Knows" summary in plain language.
3. Summarize live capabilities from `profile/capability-map.md`.
4. Summarize approval boundaries from `profile/permission-map.md`.
5. Pull the next likely power from `profile/power-map.md`, and mention the activation card if one exists.
6. List open loops that would help the next session resume faster.
7. Update `home.md` if the user's autonomy setting allows it. Otherwise show the exact proposed update.

## Output

```text
## Home Refresh

Current focus:

What changed:

What I can do now:

What needs approval:

Next power to consider:

Open loops:
```

## Rules

- Keep `home.md` short enough to scan.
- Do not turn it into a journal.
- Link out to deeper files instead of copying everything in.
- If two sections say the same thing, merge them.
