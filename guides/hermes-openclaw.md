# Hermes, OpenClaw, And Other Agent Runtimes

Lumina Agent Kit is a readable operating layer for context, memory, skills, permissions, and useful working rhythms.

Tools like Hermes, OpenClaw, and similar systems are agent runtimes. A runtime is the engine that can connect tools, channels, memory, schedules, and actions.

Lumina helps you decide what the assistant should know, how it should behave, and what boundaries it should follow. It should learn from these runtimes and stay compatible with them where possible.

## Simple Difference

Lumina:

- Helps you define goals, notes, tasks, preferences, and permissions.
- Creates a human-readable Markdown workspace.
- Teaches the assistant how to work with you.
- Gives agents portable context they can use through chat, files, and stronger runtimes.

Agent runtimes:

- Connect apps and message channels.
- Run scheduled tasks.
- Trigger tools.
- Manage more advanced automation.

## Good Order

1. Build the Lumina workspace.
2. Use it for a few real tasks.
3. Notice which tasks repeat.
4. Add tools or a runtime for repeated, useful workflows.
5. Re-run an assistant checkup after each new integration.

## What To Carry Into Any Runtime

- `home.md`
- `profile/me.md`
- `profile/goals.md`
- `profile/preferences.md`
- `profile/taste-and-standards.md`
- `profile/capability-map.md`
- `profile/power-map.md`
- `profile/permission-map.md`
- `tasks/lessons.md`
- useful skills that match your cadence
- clear yes rules

## Safety Notes

Powerful runtimes need clear boundaries.

Before adding a runtime, make sure:

- Sensitive actions require approval.
- Private files are separated from public templates.
- Secrets are stored outside normal notes.
- The assistant can explain what it can access.
