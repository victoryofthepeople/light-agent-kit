# App Access

App access means your assistant can connect to tools like Gmail, Calendar, task managers, notes apps, or project systems.

Start with the access that matches the value you need.

## Access Levels

### No Access

The assistant helps only with what you share.

Good for:

- First setup.
- Private reflection.
- Learning the system.

### Read-Only Access

The assistant can see information but cannot change the source app.

Good for:

- Calendar awareness.
- Inbox summaries.
- Reminders.
- Planning around existing commitments.

### Write Access

The assistant can create or update items.

Good for:

- Creating calendar blocks.
- Drafting tasks.
- Updating project systems.

Use write access with clear approval rules.

## Approval Rules

The assistant should ask before:

- Sending email.
- Sending messages.
- Creating calendar invites.
- Changing calendar events.
- Deleting anything.
- Publishing anything.
- Spending money.
- Using secrets or private accounts.

## Good First Step

Start with no access or read-only access.

Add write access only when the assistant already understands your preferences and clear yes rules.
