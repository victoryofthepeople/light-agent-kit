# First Week With Light Agent Kit

The first week is about proving usefulness before adding more tools.

## Day 1: Build The Map

Confirm these exist (from the guided setup, or added since):

- `home.md`
- `me.md`
- `boundaries.md`
- `tasks.md` or `tasks/active.md`
- today's file in `daily/`
- your active room (`work/`, `life/`, `project/`, or `notes/`)

Optional if you've grown the workspace: `profile/goals.md`, capability maps, `tasks/lessons.md`.

First useful action:

> Brain dump what is in your head. Ask the assistant to organize it.

## Day 2: Plan One Real Day

Use:

- `skills/morning-brief.md`
- `templates/daily/daily-template.md`

First useful action:

> Ask for a short plan with three real priorities.

## Day 3: Track One Project

Use:

- `templates/projects/project-template.md`
- `skills/project-scout.md`

First useful action:

> Pick one active project and create a simple project map.

## Day 4: Save One Lesson

Use:

- `skills/lessons-updater.md`
- `tasks/lessons.md`

First useful action:

> Correct the assistant once, then save the correction as a rule.

## Day 5: Review Access

Use:

- `profile/capability-map.md`
- `profile/power-map.md`
- `profile/permission-map.md`
- `guides/app-access.md`

First useful action:

> Decide what stays files-only, what could be read-only later, and what should always require approval. Pick the one next power worth adding.

## Day 6: Add One Skill

Use:

- `skills/README.md`
- `guides/skill-safety.md`

First useful action:

> Pick one repeatable workflow and turn it into a skill.

## Day 7: Run A Checkup

Use:

- `skills/assistant-checkup.md`

First useful action:

> Ask the assistant what is working, what is unclear, and what one upgrade would help next.

## Activation Test

By the end of week one, you should be able to say:

- What your assistant knows about you.
- What it can access.
- What it needs approval for.
- Which cadence or workflow helps.
- Which correction made it better.
- What to add next, if anything.
