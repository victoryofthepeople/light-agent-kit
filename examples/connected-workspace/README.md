# Connected Workspace Example

Fictional example.

This shows a stronger Lumina setup after a user has already proved value with files and chat.

The assistant can capture ideas from a phone channel, read calendar context, run scheduled reviews through a connected runtime, and write safe local notes. Sensitive actions still need approval.

## Scenario

Maya runs a small design studio.

She wants her assistant to:

- Capture ideas while she is away from her desk.
- Plan the day around her calendar.
- Keep client projects from scattering.
- Draft follow-ups for review.
- Run a weekly review every Friday.

## Connected Powers

- Phone capture: inbound messages become ideas or tasks.
- Calendar read-only: the assistant can see meeting times and titles.
- Local notes: the assistant can update the Lumina workspace.
- Scheduled runtime: the assistant can run morning and weekly reviews.
- Draft-only messaging: the assistant can draft follow-ups, but Maya sends them.

## Approval Boundary

The assistant still asks before:

- Sending anything.
- Publishing anything.
- Deleting anything.
- Changing the calendar.
- Using private accounts.
- Spending money.

## Files

- `home.md`: first visible summary of the connected workspace.
- `profile/capability-map.md`: what is live now.
- `profile/power-map.md`: what could grow next.
- `profile/permission-map.md`: what needs approval.
