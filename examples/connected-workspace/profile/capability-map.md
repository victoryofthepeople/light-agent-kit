# Capability Map

Fictional connected example.

What this assistant can do now. It pairs with `power-map.md` and `permission-map.md`.

## Current Mode

- [x] Files and chat only
- [x] Notes workspace
- [x] Phone access
- [x] Read-only app access
- [x] Write access with approval
- [x] Runtime automation

## Can Read

- Light Agent Kit workspace files.
- Phone messages sent directly to the assistant capture channel.
- Calendar event titles, times, and attendees.
- Project notes in the private local workspace.
- Sources the user shares for research.

## Can Suggest

- Daily plans shaped around calendar context.
- Client follow-up drafts.
- Project next steps.
- Weekly review summaries.
- Research briefs and open questions.

## Can Change With Approval

- `home.md`
- `tasks/active.md`, `tasks/done.md`
- `projects/*`
- `knowledge/*`
- `profile/*`, including these maps
- `power/*`
- Draft follow-up text saved in project notes

## Can Change Quietly

- Add phone captures to `ideas/inbox.md`.
- Update today's `daily/` file during a requested morning or evening review.
- Append to `tasks/lessons.md` after an explicit correction.
- Record weekly review notes in the dated daily file.
- Refresh `home.md` from approved workspace files.

## Connected Now

- Phone capture channel: inbound only.
- Calendar: read-only.
- Local notes: read and write inside the kit workspace.
- Scheduler runtime: morning brief and Friday review only.

## Growing Next

The next possible power is read-only email triage. It is tracked in `power-map.md` and requires approval before connecting.
