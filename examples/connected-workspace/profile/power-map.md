# Power Map

Fictional connected example.

What this assistant could grow into next. It pairs with `capability-map.md` and `permission-map.md`.

## Current Power

What the assistant can already help with:

- Capture ideas from a phone channel.
- Plan days with read-only calendar context.
- Run scheduled morning and weekly reviews.
- Organize client project notes.
- Draft follow-ups for approval.
- Turn corrections into lessons.

## Useful Next Powers

What would be valuable soon, in priority order:

- Read-only email triage for client follow-ups.
- Monthly permission audit.
- Research-to-knowledge workflow for market notes.

## Future Powers To Consider

### Messaging

- Telegram: active for inbound capture only.
- Discord: not useful right now.
- WhatsApp: maybe later for client capture, high privacy bar.
- Email: read-only triage next; sending remains approval-only.

### App Access

- Calendar: read-only active.
- Notes: local Markdown active.
- Tasks: Markdown active.
- Drive: project-specific read-only later.
- CRM or project tools: maybe later for client status.

### Scheduled Work

- Morning brief: active.
- Evening review: available when requested.
- Weekly review: active on Friday.
- Monitoring: not active.
- Follow-up reminders: useful next.

### Research

- Research reports: active on request.
- YouTube and transcript research: useful for tool and market research.
- Reddit or community research: useful for market language.
- Source-backed knowledge base: active for selected projects.

### Skills

Repeatable workflows the assistant should learn:

- Client follow-up prep.
- Weekly project scout.
- Research-to-knowledge.
- Permission audit.

### Runtime

Possible runtime direction:

- Claude/Codex: current workspace editing.
- Hermes: possible later for broader messaging and scheduled automations.
- OpenClaw: possible later for local workspace operations.
- Local: active for Markdown.
- VPS or always-on: only if scheduled work becomes frequent.

## Approval Gates

Every power above stays gated until approved. The binding gates live in `permission-map.md`.

## Next Activation

The single next power to turn on:

Read-only email triage.

Why this one:

Client follow-ups are valuable and easy to miss.

What the assistant can do:

Summarize unread client threads, suggest follow-ups, and add approved tasks to the Lumina workspace.

What it can access:

Email metadata and message text for approved accounts only.

Which approval rule applies:

Read-only first. Draft-only for replies. Sending always requires approval.
