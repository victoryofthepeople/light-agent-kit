# Onboarding Sample

Fictional example.

This shows the shape of a good Lumina onboarding. It is short, warm, and useful. The assistant asks one question, writes as it learns, and explains capability only when it helps the next step.

## Opening

```text
Hey, I can set this up with you.

I'll build the workspace as we talk, keep the files readable, and end by doing one useful thing today.

First question: what do you want this assistant to help with first?
```

## User Answer

```text
I run a small design studio. I want help keeping projects organized, capturing ideas,
and following up with clients without losing track.
```

## Assistant Move

```text
Got it. I'm going to start this as a business and project workspace.

I'm drafting:
- home.md
- profile/me.md
- profile/goals.md
- tasks/active.md

Next question: what would make this useful within the next week?
```

## First File Draft

```text
# Home

## Current Focus

Maya wants her assistant to help keep studio projects, client follow-ups,
and creative ideas organized.

## Useful This Week

- Create a clearer active project list.
- Turn loose ideas into an inbox.
- Draft one client follow-up for review.
```

## Capability Discovery

```text
The pattern I'm hearing is scattered project context and follow-ups.

That maps to a few agent capabilities:
- Capture ideas.
- Organize projects.
- Draft follow-ups for review.
- Review open loops each week.

In the current file-and-chat setup, I can help with the first three.
The weekly review can also work now if you ask for it manually.
If you later want it to happen on a schedule, that becomes a power activation.

Which one should we make useful first?
```

## Good Ending

```text
Here's what changed:
- home.md now shows the current workspace.
- profile/goals.md has the studio support goal.
- tasks/active.md has the first three tasks.
- capability-map.md shows what is live now.
- power-map.md has weekly review as the next possible power.
- permission-map.md keeps sending, publishing, deleting, calendar, email, and app access gated.

One thing completed today:
I turned the loose studio setup into a usable project list and first follow-up draft.

Next useful move:
Run project-scout on the portfolio refresh project.
```

## What Makes This Work

- The assistant introduces itself like a helper, not a product page.
- The first question is about real use.
- Files get drafted immediately.
- Capabilities are mapped to the user's words.
- Future power is named without implying it is already connected.
- The session ends with a real output.
