## Core Idea

The assistant should understand access before it receives access.

## Key Points

- Read access helps the assistant notice context.
- Write access lets the assistant change things.
- Sensitive actions need approval.
- The user should be able to explain what the assistant can access.

## Useful Frameworks

- Access levels:
  What it does: separates no access, read-only, write with approval, and quiet updates.
  When to use it: before connecting apps or tools.
