# Power Map

Fictional example.

What this assistant could grow into next. It pairs with `capability-map.md` and `permission-map.md`.

## Current Power

What the agent can already help with:

- Plan the day from goals and active tasks.
- Organize project notes.
- Turn corrections into lessons.
- Search and summarize local Markdown notes shared by the user.

## Useful Next Powers

Things that would be valuable soon:

- Weekly project review.
- Research summaries for portfolio positioning.
- Follow-up reminders for client conversations.

## Future Powers To Consider

### Messaging

- Telegram: useful later for quick task capture.
- Discord: not useful right now.
- WhatsApp: not useful right now.
- Email: read-only later, sending requires approval.

### App Access

- Calendar: read-only later.
- Notes: local Markdown first.
- Tasks: Markdown first.
- Drive: project-specific only.
- CRM or project tools: not connected.

### Scheduled Work

- Morning brief: active.
- Evening review: active.
- Weekly review: useful next.
- Monitoring: not needed yet.
- Follow-up reminders: useful next.

### Research

- Research reports: useful for client positioning.
- YouTube and transcript research: useful for learning tools.
- Reddit or community research: useful for market language.
- Source-backed knowledge base: useful later.

### Skills

Repeatable workflows the agent should learn:

- Weekly project scout.
- Research brief.
- Client follow-up prep.

### Runtime

Possible runtime direction:

- Claude/Codex: current best fit.
- Hermes: maybe later for phone and scheduled automations.
- OpenClaw: maybe later for workspace operations.
- Local: current.
- VPS or always-on: not needed yet.

## Clear Yes Rules

Every power above waits for a clear yes before it becomes active. The working rules live in `permission-map.md`.
