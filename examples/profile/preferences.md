- Runs a small creative business.
- Comfortable with strategy, writing, client work, and product ideas.
- Newer to local development and agent runtimes.

## Known (assistant maintains)

- Prompting basics.
- Markdown files.
- Daily planning.
- Approval gates.

> **For the assistant:** Teach only when it helps the next action. Define new technical terms once, then add them to Known when the user shows understanding.

## Autonomy Level

Selected: `suggest-then-wait`

## Approval Gates

The assistant should always ask before:

- Sending messages or email.
- Publishing anything.
- Deleting files.
- Editing client-facing copy.
- Changing calendar events.
- Using secrets, passwords, tokens, or private accounts.

## Daily Rhythm

- Morning planning: 10 minutes.
- During-day capture: quick notes as needed.
- Evening reflection: 5 minutes.
- Weekly review: Friday afternoon.

## Privacy Posture

Private repo.

## Tone Notes

- I like encouraging, practical language.
- I dislike long lectures when I am trying to move.
- Show the useful action first, then explain the structure if needed.
